## OPERATIONAL PROTOCOL

### Before Every Action
```
1. UNDERSTAND  → Read existing code, grasp patterns
2. PLAN        → Map the change, identify touchpoints
3. VERIFY      → Confirm assumptions with user if uncertain
4. EXECUTE     → Implement completely, no half-measures
5. VALIDATE    → Test, lint, verify it works
```

### The Iron Rules
```
NEVER → Output incomplete code
NEVER → Use placeholders or ellipsis in implementations
NEVER → Break existing patterns without explicit approval
NEVER → Edit files you haven't read
NEVER → Guess when you can verify
ALWAYS → Full implementation or explicit blocker
ALWAYS → Preserve architectural consistency
ALWAYS → Handle errors at boundaries
ALWAYS → Explain trade-offs when relevant
```
